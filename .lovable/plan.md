
# Убираем дублирующие subtitle из секций

На скриншоте видно, что каждая секция имеет два уровня: мелкую надпись капсом ("ПРОЕКТЫ") и крупный заголовок ("Наш опыт"). Subtitle по сути повторяет смысл заголовка и не несёт дополнительной ценности.

## Решение

Убрать все `.subtitle` надписи из секций, оставив только заголовки. Это сделает страницу чище и избавит от ощущения "воды".

### Какие секции затрагиваются

| Секция | Было (subtitle + title) | Станет (только title) |
|--------|------------------------|----------------------|
| Challenges | "Вызовы" + "Мы понимаем вызовы..." | Только "Мы понимаем вызовы..." |
| Services | "Услуги" + "Что мы делаем" | Только "Что мы делаем" |
| Process | "Процесс" + "Как мы работаем" | Только "Как мы работаем" |
| Projects | "Проекты" + "Наш опыт" | Только "Наш опыт" |
| Team | "Команда" + "Эксперты с опытом" | Только "Эксперты с опытом" |
| Testimonials | "Отзывы" + "Что говорят клиенты" | Только "Что говорят клиенты" |
| FAQ | "FAQ" + "Частые вопросы" | Только "Частые вопросы" |
| Contact | "Контакты" + "Давайте построим..." | Только "Давайте построим..." |

Hero остаётся без изменений — там subtitle ("Маркетинговая и IT-команда") несёт уникальный смысл.

## Технические детали

Из каждого компонента секции удаляется блок с `<p>` тегом, содержащим `t("xxx.subtitle")`. Затрагиваемые файлы:

- `src/components/ChallengesSection.tsx`
- `src/components/ServicesSection.tsx`
- `src/components/ProcessSection.tsx`
- `src/components/ProjectsSection.tsx`
- `src/components/TeamSection.tsx`
- `src/components/TestimonialsSection.tsx`
- `src/components/FAQSection.tsx`
- `src/components/ContactSection.tsx`

Ключи переводов в `LanguageContext.tsx` можно оставить (не мешают), либо удалить для чистоты.
